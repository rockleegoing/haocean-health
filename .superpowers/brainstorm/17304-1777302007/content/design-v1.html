法律法规详情页 - 章节关联依据设计

数据结构 + API 设计方案

一、数据库改动

1. 新建处理依据表 sys_processing_basis

CREATE TABLE sys_processing_basis (
    basis_id      BIGINT PRIMARY KEY AUTO_INCREMENT,
    basis_no      VARCHAR(20),           -- 编号
    title         VARCHAR(255) NOT NULL, -- 标题
    violation_type VARCHAR(100),          -- 违法类型
    clauses       TEXT,                   -- 条款内容(JSON数组)
    legal_liability TEXT,                -- 法律责任
    discretion_standard TEXT,            -- 裁量标准
    regulation_id BIGINT,                -- 关联法规
    status        CHAR(1) DEFAULT '0',
    del_flag      CHAR(1) DEFAULT '0',
    ...
);

2. 新建关联表 sys_basis_chapter_link

CREATE TABLE sys_basis_chapter_link (
    link_id      BIGINT PRIMARY KEY AUTO_INCREMENT,
    basis_type   VARCHAR(20),   -- 'legal' 或 'processing'
    basis_id     BIGINT,        -- sys_legal_basis 或 sys_processing_basis 的 ID
    chapter_id   BIGINT,        -- 关联章节
    create_time  DATETIME,
    UNIQUE KEY uk_basis_chapter (basis_type, basis_id, chapter_id)
);

四、UI交互设计

章节行点击交互

▶ 第二章 基本医疗卫生服务

交互说明：点击"定性依据 3"或"处理依据 2"徽章 → 弹出底部弹窗显示该章节关联的依据列表

方案优点

  • 数据结构清晰，关联表设计灵活
  • SQL统计查询高效（单表JOIN+COUNT）
  • 前端改动小，仅加字段和弹窗
  • 支持一个依据关联多个章节

需确认

  • 处理依据表结构是否与定性依据一致？
  • 是否需要后端同步支持？
