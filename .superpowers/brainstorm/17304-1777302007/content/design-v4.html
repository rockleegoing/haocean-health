二、数据库改动

1. sys_processing_basis 表（处理依据）

CREATE TABLE sys_processing_basis (
    basis_id          BIGINT PRIMARY KEY AUTO_INCREMENT,
    basis_no          VARCHAR(20),
    title             VARCHAR(255) NOT NULL,
    violation_type    VARCHAR(100),
    issuing_authority VARCHAR(100),
    effective_date    VARCHAR(20),
    legal_level       VARCHAR(20),
    clauses           TEXT,
    legal_liability   TEXT,
    discretion_standard TEXT,
    regulation_id     BIGINT,
    status            CHAR(1) DEFAULT '0',
    del_flag          CHAR(1) DEFAULT '0',
    ...
);

2. sys_basis_chapter_link 表（关联表）

CREATE TABLE sys_basis_chapter_link (
    link_id      BIGINT PRIMARY KEY AUTO_INCREMENT,
    basis_type   VARCHAR(20) NOT NULL,  -- 'legal' 或 'processing'
    basis_id     BIGINT NOT NULL,
    chapter_id   BIGINT NOT NULL,
    create_time  DATETIME,
    UNIQUE KEY uk_basis_chapter (basis_type, basis_id, chapter_id),
    INDEX idx_chapter (chapter_id)
);

三、后端API

1. 章节关联依据统计

GET /system/regulation/chapterBasisCount/{regulationId}

Response:
{
  "code": 200,
  "data": [
    {
      "chapterId": 1,
      "chapterNo": "第一章",
      "chapterTitle": "总则",
      "legalBasisCount": 3,
      "processingBasisCount": 2
    }
  ]
}

2. 获取某章节关联的定性依据列表

GET /system/regulation/{regulationId}/legalBasisByChapter/{chapterId}

Response:
{
  "code": 200,
  "rows": [
    {

      "basisNo": "1.",              -- 显示格式：1. 2. 3.

  ],
