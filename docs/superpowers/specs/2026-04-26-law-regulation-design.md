# 法律法规模块完善方案

## 一、整体架构

### 1.1 模块结构

```
法律法规模块
├── 后台管理端 (Vue)
│   ├── 法规管理（增删改查、导入导出）
│   ├── 章节管理（CRUD）
│   ├── 条款管理（CRUD）
│   └── 定性依据管理（增删改查）
├── 移动端 (Android)
│   ├── 法规列表（搜索、筛选）
│   ├── 法规详情（树状结构）
│   ├── 定性依据列表

│   └── 数据同步（混合模式）
```

## 二、后台管理端完善

### 2.1 分页功能修复

**问题**：章节/条款列表后端未支持分页参数

**修改文件**：
- `SysRegulationController.java` - 添加分页参数
- `ISysRegulationService.java` - 添加分页方法签名
- `SysRegulationServiceImpl.java` - 实现分页逻辑
- `SysRegulationMapper.java` / `SysRegulationMapper.xml` - 添加分页SQL

**修改后接口**：
```
GET /system/regulation/chapters/{regulationId}?pageNum=1&pageSize=10
GET /system/regulation/articles/{regulationId}?pageNum=1&pageSize=10&chapterId=1
```

### 2.2 批量导入/导出功能

**导入功能**：
- 支持 Excel (.xlsx) 和 JSON 两种格式
- 导入时数据校验（必填项、格式校验）
- 导入结果提示（成功数、失败数、错误详情）

**导出功能**：
- Excel 导出：使用 Apache POI 库
- JSON 导出：直接序列化
- 支持按条件导出（法规类型、状态等）

**导入模板**：
```json
{
  "regulations": [
    {
      "title": "法律名称",
      "legalType": "法律",
      "supervisionTypes": ["医疗机构"],
      "publishDate": "2020-01-01",
      "effectiveDate": "2020-06-01",
      "issuingAuthority": "全国人民代表大会常务委员会",
      "content": "完整内容",
      "chapters": [
        {
          "chapterNo": "第一章",
          "chapterTitle": "总则",
          "sortOrder": 1,
          "articles": [
            {"articleNo": "第一条", "content": "条款内容", "sortOrder": 1}
          ]
        }
      ]
    }
  ]
}
```

### 2.3 法规详情增强

**新增功能**：
- 法规详情页面添加"管理章节"按钮
- 跳转章节管理并自动选中该法规

## 三、移动端完善

### 3.1 离线同步机制（混合模式）

**同步流程**：
```
首次同步：
  1. 检查本地数据库是否为空
  2. 全量下载法规、章节、条款、定性依据
  3. 记录同步时间到 SharedPreferences

增量同步：
  1. 调用接口获取 updateTime > 上次同步时间 的数据
  2. 差异更新本地数据库
  3. 更新同步时间
```

**同步触发时机**：
- App 启动时（后台静默同步）
- 下拉刷新列表时
- 用户手动点击同步按钮

### 3.2 法规详情页面（树状图）

**UI 结构**：
```
┌─────────────────────────────────┐
│ ← 返回    法律名称         ⋮  │
├─────────────────────────────────┤
│ 发布日期：2020-01-01            │
│ 实施日期：2020-06-01            │
│ 颁发机构：全国人民代表大会...    │
├─────────────────────────────────┤
│ [法定依据] [定性依据]           │
├─────────────────────────────────┤
│ ▼ 第一章 总则                    │
│   · 第一条 条款内容...           │
│   · 第二条 条款内容...           │
│ ▶ 第二章 基本医疗卫生服务        │
│ ▶ 第三章 医疗机构                │
└─────────────────────────────────┘
```

**交互行为**：
- 点击章节标题：展开/折叠条款列表
- 点击条款：跳转条款详情页
- 列表支持滚动优化（大列表使用 RecyclerView）

### 3.3 定性依据详情页面

**新增功能**：
- 条款内容复制按钮（复制到剪贴板）
- 裁量标准分色展示（轻微/一般/严重）

- `sys_collection` 表相关代码（收藏功能）

## 四、数据流设计

### 4.1 同步时序图

```
┌─────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────┐
│  App   │     │ Repository  │     │   LawApi    │     │ Server  │
└───┬────┘     └──────┬──────┘     └──────┬──────┘     └───┬─────┘
    │                  │                   │                │
    │ 启动/刷新        │                   │                │
    │ ──────────────> │                   │                │
    │                  │ 获取上次同步时间    │                │
    │                  │ ─────────────────>│                │
    │                  │                   │ GET /list      │
    │                  │                   │ ──────────────>│
    │                  │                   │                │
    │                  │                   │ <──────────────│
    │                  │                   │ 返回数据       │
    │                  │ 更新本地数据库     │                │
    │                  │ <────────────────│                │
    │ 返回结果         │                   │                │
    │ <─────────────── │                   │                │
    │                  │                   │                │
```

### 4.2 冲突处理

- 法律法规：服务端优先（本地不能修改）
- 定性依据：服务端优先
